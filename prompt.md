# chern_capital 项目交接与持续规则

> 给后续接手本项目的 Codex：开始修改前，请先阅读本文件，再查看当前页面与相关源文件。这个站点不是一次性演示稿，而是在逐步累积的个人交易工作台；新增功能应建立在已有内容之上。

## 项目目标

`chern_capital` 是一个个人交易网站，当前主要承载三类内容：

1. **脚本筛选**：说明 VCP 筛选与图表复核流程，保留具体规则和运行方式。
2. **实盘记录**：展示真实持仓的建仓位置、日线图、持仓天数和盈亏变化。
3. **交易阅读**：用空间记忆与关联路径的方式呈现阅读摘录和心得，第一本为《缠论》。

网站视觉应保持克制、深色、交易工作台风格；内容重点是可阅读、可复核、可继续扩展，而不是营销式首页。

## VCP 脚本关系

这一点曾经被页面摘要模糊过，后续不要再混写：

- `vcp_screener.py` 负责扫描股票池并执行筛选规则，输出 `vcp_result_YYYYMMDD.csv`。
- `vcp_plot.py` 读取最新筛选结果，拉取前复权日线，生成可交互复盘页面 `vcp_charts_YYYYMMDD_HHMM.html`。
- 两者共同构成“先筛，再看图”的流程；`vcp_plot.py` 不是完整筛选规则本身。

当前需要在网站中保留的筛选条件：

1. 股票池限定为代码以 `60` 或 `00` 开头的沪深主板股票，排除 ST 与退市标记样本。
2. 周线满足 `Close > MA10 > MA30`，且周 MA10、MA30 相比 4 周前均向上。
3. 收盘价高于日线 MA50，距 52 周高点回撤在 `(3%, 25%]` 内，并高于 52 周低点的 `1.3` 倍。
4. 近 5 日平均成交量低于近 20 日均量的 `80%`：`vol5 / vol20 < 0.8`。
5. 近 5 日平均振幅低于近 20 日平均振幅的 `80%`：`amp5 / amp20 < 0.8`。
6. 最近 5 日中，后 3 日平均振幅低于前 2 日，表达整理尾端继续变紧。

## 内容与设计规则

以下规则优先级很高。除非用户明确要求改变，否则每次版本更新都应遵守。

### 内容保留

1. **新增功能不能以大幅压缩既有内容为代价。** 尤其不能因为新增卡片、模块或导航入口，就把已经确认过的说明压成一句摘要。
2. 修改某个板块前，先清点其已有标题、规则、数据和可访问入口；改完后检查这些内容仍存在，或被明确迁移到可点击到达的详情页。
3. 如果因层级整理需要将正文移到详情页，首页必须保留清晰入口与足够语境，且详情页应承接原有信息量。
4. 用户提供的原始摘录、持仓记录、脚本规则属于核心资料；不应自行删改、泛化或替换为空泛文案。
5. 每次产生新的结构调整或关键内容变更时，同步更新本文件的版本记录和规则说明。

### 信息层级

1. 首页以“交易实战”为第一层级，避免重新变成内容混杂的长列表。
2. “脚本筛选”和“实盘”属于交易实战下的并列模块；实盘新增不能挤掉脚本说明。
3. VCP 的流程和六条规则应继续能在首页阅读；它们不是装饰性简介。
4. “阅读”在首页保持入口性质，具体摘录与记忆宫殿体验放在 `reading.html`。
5. 未来新增其他实盘交易或其他书籍时，延续同层新增记录/书架项的结构，不覆盖现有第一条内容。

### 视觉与交互

1. 延续当前深色、克制、偏交易终端/工作台的风格，不改造成营销落地页。
2. 阅读宫殿必须包含空间感、视觉符号与关联路径，不能退化为只有文字说明的列表。
3. 实盘详情应以日线图为主要叙事载体，保留建仓线、成交记录、持仓指标与盈亏说明。
4. 所有新增界面必须检查桌面与手机布局，避免文字遮挡、日期刻度重叠或横向滚动。
5. 数据刷新等操作必须给访问者明确状态反馈；失败时保留已有快照并说明原因。
6. 实盘详情页进入 Cloudflare 版本时应自动刷新一次行情或日线数据，并保留手动刷新按钮；GitHub Pages 静态版本不自动发起必定失败的接口请求。
7. 已结束交易应展示买入点、卖出点、净回收和已实现盈亏；不要继续把它写成“当前持仓”或“浮动盈亏”。

### 数据与发布安全

1. 不将行情展示表述为投资建议，不替用户生成买卖结论。
2. 用户的真实交易数据可以按其明确提供的内容展示；添加新交易前核对日期、数量、成交价与成本口径。
3. 新浪即时行情当前通过 Cloudflare Function 请求，并需要服务端设置来源头；修改 API 前先验证线上数据请求是否仍可用。
4. 不把密钥、令牌、登录状态或本地部署缓存提交到仓库。
5. `vcp/`、`read/` 与 `.wrangler/` 当前被 `.gitignore` 排除；除非用户明确同意，不上传原始脚本、原始阅读文件或部署缓存。
